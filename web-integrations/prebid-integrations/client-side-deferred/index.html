Deferred ${IDENTITY_NAME} Integration with Prebid.js

This example demonstrates how to integrate ${IDENTITY_NAME} with Prebid.js using deferred configuration. Unlike the standard integration where ${IDENTITY_NAME} is configured on page load, this pattern uses mergeConfig() and refreshUserIds() to add ${IDENTITY_NAME} after the page has loaded.   1. pbjs.mergeConfig(uidConfig) - Merges ${IDENTITY_NAME} user ID module configuration into the existing Prebid config without overwriting other settings. This allows adding ${IDENTITY_NAME} after the page has already loaded.

What's Happening?

          
    
            
  • Initial Load: Prebid initializes with setConfig() but no ${IDENTITY_NAME} module configured yet
    • 
            
  • Deferred Config: When user provides email, mergeConfig() adds ${IDENTITY_NAME} without overwriting existing settings
    • 
            
  • Token Generation: refreshUserIds() triggers Prebid to generate the ${IDENTITY_NAME} token using CSTG credentials
    • 
            
  • Local Storage: Prebid stores tokens in localStorage (${UID_STORAGE_KEY}) for persistence across page loads
